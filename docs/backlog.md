# Backlog del Modelo Integral de Autonomía

Este documento reúne todas las ideas, módulos y funcionalidades que aún no forman parte de la versión actual del proyecto.

Su objetivo es evitar perder ideas sin interrumpir el trabajo en curso.

## Reglas

- Una idea nueva no implica crear un documento nuevo.
- Una idea nueva no modifica la arquitectura actual.
- Toda idea primero ingresa al backlog.
- El backlog se revisa al planificar una nueva versión del modelo.
