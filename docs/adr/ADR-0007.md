# ADR-0007 — Desarrollar el modelo mediante iteraciones y documentación continua

**Estado:** Aceptada

**Versión del proyecto:** 0.1.0

**Autoría inicial:** Natalia Moroni

**Redacción y documentación:** ChatGPT (OpenAI) + Natalia Moroni

**Fecha:** 2026-07-02

---

# Contexto

El Modelo Integral de Autonomía aborda un problema complejo que involucra dimensiones sociales, económicas, educativas y comunitarias.

Intentar diseñar desde el inicio una solución completa implicaría un alto riesgo de parálisis, pérdida de foco y dificultad para validar las ideas.

Durante el desarrollo del proyecto se decidió adoptar una metodología inspirada en el desarrollo de software, donde el modelo evoluciona mediante versiones sucesivas, decisiones documentadas e incorporación gradual de nuevos módulos.

---

# Alternativas consideradas

## Opción A — Diseñar el modelo completo antes de comenzar

Buscar una versión definitiva del sistema antes de iniciar cualquier implementación.

### Motivo para descartarla

En problemas complejos es imposible prever todas las variables desde el comienzo.

Además, este enfoque dificulta el aprendizaje continuo y retrasa la validación de las ideas.

---

## Opción B — Desarrollar el modelo de manera iterativa

Construir el sistema mediante pequeñas versiones funcionales, documentando las decisiones y permitiendo su evolución a medida que se obtiene nueva información.

### Resultado

**Opción elegida.**

---

# Decisión

El Modelo Integral de Autonomía será desarrollado de forma iterativa.

Cada versión incorporará mejoras, nuevos módulos o ajustes sobre la base de la experiencia y del aprendizaje obtenido durante el proceso.

Las decisiones arquitectónicas relevantes serán documentadas mediante ADR y el trabajo pendiente se organizará mediante un backlog priorizado.

---

# Fundamentación

La complejidad del proyecto requiere una metodología que favorezca el aprendizaje continuo y la mejora progresiva.

Este enfoque permite comenzar con objetivos alcanzables, validar hipótesis y reducir el riesgo de construir soluciones innecesarias o inviables.

---

# Consecuencias

Esta decisión implica que:

- El proyecto tendrá versiones claramente identificadas.
- Se mantendrá un backlog priorizado.
- La documentación evolucionará junto con el modelo.
- Se aceptará que algunas decisiones cambien a medida que el conocimiento del sistema aumente.

---

# Preguntas abiertas

- ¿Cómo se definirá el contenido de cada versión?
- ¿Qué criterios determinarán que una versión está finalizada?
- ¿Cómo se gestionarán los aportes de futuros colaboradores?

Estas preguntas serán desarrolladas en futuras versiones del modelo.
