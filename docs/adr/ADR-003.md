# ADR-0003 — Diseñar un modelo modular y replicable

**Estado:** Aceptada

**Versión del proyecto:** 0.1.0

**Autoría inicial:** Natalia Moroni

**Redacción y documentación:** ChatGPT (OpenAI) + Natalia Moroni

**Fecha:** 2026-07-02

---

# Contexto

Muchos proyectos sociales se diseñan para responder a las necesidades de una comunidad específica y dependen fuertemente de las personas que los impulsan, del contexto local o de recursos particulares.

Esto dificulta que otras organizaciones puedan adoptar el modelo, adaptarlo o replicarlo en otros territorios.

Desde su concepción, el Modelo Integral de Autonomía busca trascender la creación de una única institución.

El objetivo es desarrollar un sistema que pueda ser implementado, adaptado y ampliado por diferentes organizaciones, respetando las particularidades de cada comunidad.

---

# Alternativas consideradas

## Opción A — Diseñar un proyecto único

Construir un modelo pensado exclusivamente para una organización y un territorio determinados.

### Motivo para descartarla

Limita la posibilidad de que otras comunidades adopten el modelo y reduce el impacto potencial del proyecto.

---

## Opción B — Diseñar un modelo modular y replicable

Construir una arquitectura compuesta por módulos relativamente independientes que puedan implementarse de manera gradual y adaptarse a diferentes realidades.

### Resultado

**Opción elegida.**

---

# Decisión

El Modelo Integral de Autonomía será diseñado como un conjunto de módulos funcionales que podrán implementarse de manera independiente o integrada.

Cada módulo deberá definir claramente:

- su propósito;
- sus requisitos mínimos;
- las relaciones con otros módulos;
- los recursos necesarios;
- los indicadores para evaluar su funcionamiento.

---

# Fundamentación

La modularidad facilita el crecimiento progresivo del sistema.

Una organización podrá comenzar implementando uno o pocos módulos y, a medida que disponga de recursos, incorporar otros sin necesidad de rediseñar el modelo completo.

Este enfoque también favorece la adaptación a distintos contextos sociales, económicos y culturales.

El objetivo no es que todas las implementaciones sean idénticas, sino que compartan una arquitectura común.

---

# Consecuencias

Esta decisión implica que:

- Ningún módulo deberá depender innecesariamente de otro.
- Cada módulo deberá documentarse de forma independiente.
- Será posible implementar versiones parciales del modelo.
- El crecimiento del sistema podrá realizarse de manera incremental.

---

# Preguntas abiertas

- ¿Cuál será el conjunto mínimo de módulos para considerar una implementación válida del modelo?
- ¿Qué módulos serán obligatorios y cuáles opcionales?
- ¿Cómo se documentarán las dependencias entre módulos?

Estas preguntas serán desarrolladas en futuras versiones del modelo.
